# 隐私政策

**生效日期：** 2026 年 7 月 16 日

本隐私政策说明由 ChanTinPing（“发布者”）发布和维护的 Auto-Optimize Codex AGENTS.md（“本 Skill”）如何处理数据。本政策适用于本 Skill 随附的指令、脚本及发布者维护的支持渠道。OpenAI、GitHub、Git、操作系统及其他服务分别适用其自身的条款与隐私规则。

## 摘要

发布者没有为本 Skill 运营后端，也不会自动收到你的 Codex 历史、优化器产物或已学习规则。随附的 Python 脚本不包含 HTTP 客户端、遥测、分析、广告或直接上传功能。但本 Skill 会读取本地会话内容，并把其中选定内容复制到本地运行和状态文件；当这些内容进入 Codex 上下文时，Codex 可能会根据你的 OpenAI 产品、账户、工作区及数据控制设置进行处理。

## 本 Skill 处理的数据

根据具体工作流，本 Skill 可能读取或处理：

- Codex 的活跃及归档会话 JSONL，包括用户输入与 Assistant 最终回复；
- 会话、线程、轮次及记录标识符，时间戳，活跃/归档状态，来源路径，行与字节位置，工作目录及推断出的项目根目录；
- 用于识别真实用户轮次、完整回复、注入上下文和子代理会话的事件结构；
- 按需读取的证据片段，包括用户或 Assistant 消息、生命周期事件，以及工具名称、参数、输入或输出摘要；
- 计划任务运行前配额检查使用的 token 配额及速率限制观测；
- 现有项目级与全局 `AGENTS.md` 内容；
- 优化器状态、历史决策、已拒绝候选、撤回墓碑、报告及游标；
- 本地 Git 状态、索引元数据、仓库根目录及提交标识符。

工具摘要和会话内容可能包含源代码、文件内容、个人信息、凭据或秘密。证据工具会限制事件类型和长度，但不会检测或清除敏感内容。

## 使用目的

本 Skill 使用上述数据来重建已完成对话、识别长期工作偏好、判断项目或全局作用域、生成带证据的建议与 diff、防止重复规则或已撤回规则重新出现、应用经过授权的托管区块变更，以及维护增量检查点。

## 本地存储与修改

文档所述工作流会在 `$CODEX_HOME/auto-optimize-codex-agents-md/` 下保存明文产物，包括按时间组织的运行目录、扫描清单、批次、审查分块、可选证据、决策、计划、`state.json`、`decisions.jsonl`、报告和锁文件。这些文件可能包含完整的选定用户输入与 Assistant 最终回复、绝对路径、候选规则、证据引用、diff 及 Git 提交 ID。本 Skill 不提供应用层静态加密。

Suggest 模式不会修改目标 `AGENTS.md`，但会写入优化器状态、审计、报告和运行产物。已确认 Suggest 与经过明确授权的 Auto 工作流可以创建或修改已授权项目根目录或全局 `AGENTS.md` 中的托管区块。根据模式和已保存偏好，本 Skill 可以创建只包含目标 `AGENTS.md` 的本地 Git 提交；它不会运行 `git push`。用户配置的 Git hook 或签名辅助程序可能产生不受本 Skill 控制的额外效果。

随附脚本以只读方式打开原始 Codex 会话 JSONL，不会修改或删除这些文件。

## 网络处理与第三方

随附脚本不会直接通过网络传输数据，发布者也没有接收运行数据的服务。但这不表示完整工作流完全离线：由 Codex 审查的内容可能按照适用于你的账户或工作区的 OpenAI 条款与数据控制进行处理。计划任务定义和运行元数据可能由 Codex 的计划任务功能处理。GitHub 会按照其自身政策处理仓库访问、Issue、安全报告及其他支持活动。

## 保留与删除

本 Skill 没有自动到期、日志轮换或安全擦除功能。本地运行产物、状态、报告、审计记录及已复制的会话文本会一直保留，直到你或外部清理流程将其移除。拒绝建议或撤回规则不会清除相关审计记录、报告、墓碑、运行文件或 Git 历史。

如需移除优化器在本地保留的数据，请另外停用相关计划任务、移除不再需要的已学习规则，并移除 `$CODEX_HOME/auto-optimize-codex-agents-md/` 优化器目录。移除该目录会删除检查点，并可能导致仍存在的来源会话被重新处理。本地 Git 提交及任何已经推送到远端的历史必须分别管理。删除原始 Codex 会话不会自动删除已经复制到优化器产物中的文本。

## 通过支持渠道收到的数据

如果你创建 GitHub Issue 或通过其他方式联系发布者，发布者会收到你主动提交的信息。请勿在公开 Issue 中加入原始会话全文、凭据、秘密、个人数据或未脱敏的绝对路径。涉及安全的问题应使用 [SECURITY.md](SECURITY.md) 所述的私密渠道。

## 安全

本 Skill 使用限定路径、托管区块写入、原子本地文件、证据范围限制及明确授权门控来降低风险，但任何软件都无法保证绝对安全。你有责任管理文件系统权限、设备安全、备份、OpenAI 数据控制、Git 配置，并在接受前审查变更。

## 政策变更与联系

本政策可能随 Skill 行为或提交要求变化而更新。重大变更会在本仓库公布。如有隐私问题，请使用私密安全报告渠道或 [SUPPORT.zh-CN.md](SUPPORT.zh-CN.md) 中列出的方式。
