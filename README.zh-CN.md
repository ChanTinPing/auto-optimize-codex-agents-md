使用 Codex 时，我们常会发现它没有做到一些我们认为理所当然的事：进行过多不必要的测试、对项目出现违反常理的理解偏差。即使我们在当前对话中纠正它，一旦开启新对话，纠正往往又会被遗忘。一个合理的 harness 应当能从这些互动中持续吸收用户的审美、倾向与风格。本 Skill 正是为此而生：它自动分析 Codex 的聊天记录，提炼长期可复用的偏好，并将其写入项目级或全局 `AGENTS.md`，让 Codex 具备自动学习能力（也可以理解为一个极简版 Hermes）。本项目刻意只学习适合写入 `AGENTS.md` 的偏好，而不考虑 Skill：我们认为不应限制 Agent 的实践手段，因为这类约束在某些情境下反而会成为累赘，如 LLM 升级；相比之下，用户的偏好不会随着 LLM 的升级而自然被学会，因此才是值得长期保存的部分。

下载本 Skill 后，可以单独开启一个新对话（或新建一个专门项目），告诉 Codex“请根据此 Skill 优化 `AGENTS.md`”。Codex 会挖掘历史记录并给出带编号的建议；你只需说明接受哪些建议，它便会自动完成相应优化。首次整理完成后，还可以让 Codex 配置定时任务，持续挖掘新增的历史记录并提出增量改进。

# Auto Preference Learner

## 功能

- 挖掘 Codex 的活跃及归档会话历史，识别用户的明确纠正、同时指出问题与可执行改进的不满反馈、可复用的审美及验收标准、沟通与授权边界，以及对 `AGENTS.md` 规则的明确增删改要求。
- 将值得长期保留的行为偏好按适用范围整理为项目级或全局 `AGENTS.md` 规则。
- 首次挖掘后持续扫描新增历史，只处理尚未分析的记录，并保留暂时不足以形成全局规则的候选，供后续证据补全。
- 为每项建议保留来源证据并生成完整 diff，支持接受、编辑、拒绝、撤回和恢复已经学到的规则。

## 工作模式

### Suggest（默认）

Suggest 会完成历史挖掘和规则判断，但只向用户展示带编号的建议、原因与完整 diff，不会立即修改目标 `AGENTS.md`。用户可以自然语言说明接受哪些编号，也可以编辑或拒绝其中任何一项；只有明确接受的建议才会写入。首次使用以及需要人工把关的场景建议采用此模式。

### Auto（需明确授权）

Auto 适合定时执行无需逐项确认的增量维护，只有用户明确启用后才会运行。它只自动应用高置信度、低过度约束风险的规则；证据不足、适用范围不清或风险较高的内容不会被强行写入。对于 Git 项目，成功写入后可按用户保存的偏好只提交目标 `AGENTS.md`，不会夹带仓库中的其他改动。

## 边界与安全

本 Skill 只学习适合长期保存的 Codex 行为偏好，不提供独立的会话导出功能、不管理一般会话，也不生成工作流 Skill。它只把真实的用户输入与 Assistant 最终回复作为主要判断材料，不把系统注入、推理过程、工具输出或子代理内容误当成用户偏好。写入时只修改自己的托管区块，保留 `AGENTS.md` 其余内容；项目范围只来自可信会话元数据确定的根目录；遇到损坏的标记或符号链接目标时会拒绝写入。原始会话历史始终只读。有关本地产物及 Codex 处理方式，请参阅英文版[隐私政策](PRIVACY.md)。

## 环境要求

- Codex，并且其配置的 `CODEX_HOME` 中存在本地会话历史。
- Python 3.10 或更高版本。
- Git，用于识别项目根目录，以及按用户偏好为确认后的变更创建提交。

## 安装

安装为用户级 Skill：

```bash
git clone https://github.com/ChanTinPing/auto-optimize-codex-agents-md.git ~/.agents/skills/auto-preference-learner
```

Codex 通常会自动检测 Skill 变更。如果没有出现，请重启 Codex。
